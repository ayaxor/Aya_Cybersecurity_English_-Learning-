

# Security Policy

This repository is for **training in cybersecurity and software supply chain security**.

## 1. Scope

- Do **not** use this repository to test attacks against real systems.
- All examples must use **fake or lab environments** only.
- Do not store secrets, private keys, real tokens, or passwords here.

## 2. Reporting a Security Issue

If you discover:

- A vulnerability in example code,
- A misconfiguration in the training material,
- Or a security risk in how the repo is structured,

please:

1. Do **not** publish details publicly.
2. Open a private Issue (if available) or contact your mentor directly.
3. Provide a short description and, if possible, steps to reproduce.

## 3. Best Practices for Learners

- Never copy real company credentials into examples.
- Use strong, unique passwords in any lab.
- Treat all logs and screenshots as potentially sensitive.

This repository is for **learning to protect systems**, not for causing harm.
