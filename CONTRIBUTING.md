

# Contributing Guidelines

Thank you for your interest in contributing to this cybersecurity & English learning repository.

## 1. Purpose

This repository is used for training as part of the **QuietWire AI Training Program**.  
The main goals are:
- Practice clear technical writing (English + Arabic).
- Practice basic GitHub workflows (branches, commits, pull requests).
- Practice structuring cybersecurity learning materials.

## 2. How to Contribute

1. **Fork or create a branch** from `main`.
2. Add or update content in:
   - `English/` for English materials.
   - `Arabic/` for Arabic materials.
3. Keep documents **simple, accurate, and beginner-friendly**.
4. Submit a Pull Request (PR) with:
   - A clear title (e.g., `Add example for software supply chain risk`).
   - A short description of what you changed and why.

## 3. Style

- Use clear headings and bullet points.
- Explain technical terms in simple language first.
- When possible, provide small examples instead of long theory.
- Keep English and Arabic sections aligned in meaning.

## 4. Commit Messages

- Use short, meaningful messages:
  - `Add intro section to EN document`
  - `Fix typo in AR version`
  - `Update SECURITY policy`

## 5. Questions

If you are unsure about anything, open an **Issue** with your question, or ask your mentor.
